## 4.1 技术栈选择

•	前端框架: 纯 HTML/CSS/JavaScript —— 理由：轻量级，无需构建，部署简单，适合个人静态网站。
•	样式: 自定义 CSS —— 理由：完全可控，无额外依赖，便于维护。
•	本地存储: LocalStorage —— 理由：用于后台数据存储，无需后端服务器。
•	部署平台: Vercel —— 理由：自动化部署，支持静态网站，速度快，免费使用。

## 4.2 本地运行步骤

1.	克隆仓库:

```bash
git clone https://github.com/lxd-1/lixiaodan-personal-website.git
```

2.	本地运行:
   - 方式一：直接打开 `index.html` 文件（推荐用于快速预览）
   - 方式二：使用本地服务器（推荐用于完整功能测试）

```bash
# 使用 Python 启动本地服务器
python -m http.server 3000

# 或使用 Node.js http-server
npx http-server -p 3000

# 打开浏览器访问 http://localhost:3000
```

3.	后台管理访问:
   - 访问 `http://localhost:3000/admin/index.html`
   - 无需登录，直接进入后台管理界面

## 4.4 核心功能说明

1.	**个人信息管理**:
   - 在后台管理页面可以编辑个人基本信息
   - 支持上传和更新头像、关于我照片
   - 信息保存在浏览器 LocalStorage 中

2.	**照片上传功能**:
   - 支持上传头像（用于主页展示）
   - 支持上传关于我照片（用于关于页面）
   - 支持修改和删除照片
   - 照片会被压缩后存储，优化加载速度

3.	**响应式设计**:
   - 适配不同屏幕尺寸
   - 移动端友好的布局
   - 流畅的交互体验

## 4.5 部署说明

1.	**Vercel 自动部署**:
   - 项目已连接 Vercel，推送到 GitHub 后自动触发部署
   - 部署成功后可通过 Vercel 提供的域名访问
   - 部署地址: https://index-snowy-ten.vercel.app

2.	**手动部署**:
   - 登录 Vercel 控制台
   - 选择 "New Project"
   - 导入 GitHub 仓库 `lixiaodan-personal-website`
   - 选择 "Other" 作为框架预设
   - 点击 "Deploy" 开始部署

## 4.6 开发注意事项

1.	**文件命名**:
   - 所有文件使用英文命名，避免中文乱码问题
   - 图片文件使用小写字母和下划线命名

2.	**代码规范**:
   - HTML 遵循语义化标签规范
   - CSS 使用类选择器，避免 ID 选择器滥用
   - JavaScript 代码添加必要注释，保持可读性

3.	**浏览器兼容性**:
   - 支持现代浏览器（Chrome、Firefox、Safari、Edge）
   - 不兼容 IE 浏览器

4.	**数据管理**:
   - 所有数据存储在 LocalStorage 中，清除浏览器数据会丢失
   - 建议定期备份重要数据
